창업을 한후 시간이 좀 지났지만, 미루고 미루던 블로그를 GatsbyJS와 Github Pages를 이용하여 만들어 보았다.

## GatsbyJS란?
Gatsbyjs는 React 기반의 정적 페이지 생성 프레임워크로 가장 큰 특징은 Graphql로 사이트 내에 여러정보를 쿼리하여 그 데이터를 HTML, CSS, React로 만들어진 컴포넌트 UI에 뿌릴 수 있다.

GatsbyJS를 선택한 이유는 생태계가 잘 구성되어있고 Gitbub Pages를 이용하여 호스팅을 할 수 있기 때문이다.
많은 분들이 [gatsby-starter-bee](https://github.com/JaeYeopHan/gatsby-starter-bee)를 추천하고 있는데, 현재 버전의 nodejs(v21.3.0)에선 의존성 해결이 어려워 보여서, GatsbyJS 공식 사이트에 있는 많은 스타터 템플릿 중 내가원하는 레이아웃과 유사하고, 다크모드, SEO(Search Engine Optimization)가 이미 구현되어있는  [gatsby-starter-minimal-blog](https://github.com/LekoArts/gatsby-starter-minimal-blog)를 이용하여 구현하였다.

맥을 이용하여 구현하였기 때문에, 맥을 이용하시는 분들을 위한 설명이다.

먼저 nodejs를 설치한다.
```bash
brew install node
```

그리고 gatsby-cli를 설치한다.
```bash
npm install -g gatsby-cli
```

다음으로 gatsby-starter-minimal-blog 템플릿을 이용하여 블로그를 생성한다.
```bash
npx gatsby new gatsby-starter-minimal-blog https://github.com/LekoArts/gatsby-starter-minimal-blog
```

이후 생성된 폴더로 이동하여, gatsby develop을 실행한다.
```bash
cd gatsby-starter-minimal-blog
gatsby develop
```

그럼 아래와 같이 블로그가 실행된다.
```bash
⠀
You can now view minimal-blog in the browser.
⠀
  http://localhost:8000/
⠀
View GraphiQL, an in-browser IDE, to explore your site's data and schema
⠀
  http://localhost:8000/___graphql
⠀
Note that the development build is not optimized.
To create a production build, use gatsby build
⠀
success Building development bundle - 1.876s
success Writing page-data.json and slice-data.json files to public directory - 0.042s - 1/6 143.90/s
```

브라우저에서 http://localhost:8000 로 접속하면 [개발자 라이브 데모](https://minimal-blog.lekoarts.de/)와 동일한 화면을 볼 수 있다.
